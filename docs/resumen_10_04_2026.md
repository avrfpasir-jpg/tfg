# Resumen de Cambios (10/04/2026)

Este documento resume todas las operaciones e implementaciones llevadas a cabo el 10 de abril de 2026 en la infraestructura del proyecto SENTINEL, centrado en automatización de monitorización, auto-scaling y proxy de red.

## 1. Auto-Discovery de Servidores Web en Prometheus
Se modificó el archivo de configuración `prometheus.yml` en la máquina de monitorización para hacerlo dinámico y compatible con Auto-Scaling:
- **Redes / IPs estáticas eliminadas**: Se sustituyeron los targets correspondientes a los `web-node` y `web-apache` que usaban IPs fijas por reglas automáticas de auto-descubrimiento en AWS (`ec2_sd_configs`).
- **Filtrado por Etiquetas**: Prometheus buscará las instancias nuevas en AWS basándose en la etiqueta `Name` con regex `.*Sentinel.*` dentro de la región de AWS, asignándoles el rol de `web`.

## 2. Automatización DNS mediante Cron Jobs (DuckDNS)
Se subieron e instalaron los scripts encargados de mantener sincronizados los dominios DuckDNS con las IPs de la infraestructura actual:
- Se ajustó la variable `DOMAIN="statuspsicopompo"` dentro de la plantilla `update_duckdns_status.sh` para hacerla coincidir con el dominio real provisto.
- **Ruta de scripts**: Copiados a `/home/ubuntu/scripts/` en el servidor de monitorización.
- **Ruta de logs**: Guardado de salida de ejecución en `/home/ubuntu/logs/`.
- **Crontab**: Modificación de las tareas programadas (`cron`) del sistema añadiendo dos reglas de ejecución que disparan la comprobación DNS cada 5 minutos:
  ```cron
  */5 * * * * /home/ubuntu/scripts/update_duckdns_status.sh >> /home/ubuntu/logs/duckdns_status.log 2>&1
  */5 * * * * /home/ubuntu/scripts/update_duckdns.sh >> /home/ubuntu/logs/duckdns.log 2>&1
  ```

## 3. Guía de Proxy Inverso Nginx para Grafana
Para evitar el uso obligatorio de exponer y adjuntar el puerto específico (`:3000`) en la URL al intentar acceder a los reportes y tableros (e.g. `statuspsicopompo.duckdns.org:3000`), se diseñaron y documentaron los pasos de un pasarela transparente mediante proxy:
- **Flujo de tráfico**: Instalar `nginx` para que las peticiones al puerto HTTP (`80`) de dicha máquina las re-escriba y canalice internamente hacia el servidor `grafana` (`localhost:3000`).
- **Seguridad**: Instrucciones para la actualización paralela de los Security Groups en Amazon EC2 (permitiendo conexiones `HTTP` procedentes de `0.0.0.0/0`).
